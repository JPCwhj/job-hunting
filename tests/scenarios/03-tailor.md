# Tailor 验证场景

## 验证检查点（每个生成的三件套都要检查）

### resume.md
- [ ] 与主简历段落结构一致，不多不少（没有凭空多出来的经历或技能）
- [ ] 没有出现主简历中从未提到过的公司、项目、技能
- [ ] 有数字出现时，数字与主简历一致（没有被修改或放大）
- [ ] 若有 `[请填写：xxx]`，旁边有清晰描述说明需要填什么
- [ ] 若有 `[需用户确认]`，标注了具体的推断内容

### opener.md
- [ ] 字数 ≤ 200 字（汉字计数）
- [ ] 提到的经历可以在主简历中找到原文依据
- [ ] 没有引用含 `[请填写：xxx]` 的具体数字
- [ ] 结构清晰：自我介绍 + 相关经历 + 沟通意愿

### changelog.md
- [ ] 每一条改动都有「原因」说明
- [ ] 所有在 resume.md 中的改动，changelog 都有对应记录（双向比对）
- [ ] 没有漏记改动（改动数量与 resume.md 改动量大致吻合）
- [ ] 需用户回填项和需用户确认项都在 changelog 中有标注
